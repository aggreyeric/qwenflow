# Security Policy

## Supported Versions

Only the latest release of QwenFlow receives security fixes.

| Version | Supported          |
|---------|--------------------|
| latest  | ✅ Yes             |
| older   | ❌ No (upgrade)    |

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security problems.

Instead, report vulnerabilities privately by emailing **security@qwenflow.dev**.
Include a clear description and, if possible, steps to reproduce the issue.
We will review every report and respond as soon as we can.

## Scope

We appreciate reports for genuine vulnerabilities in QwenFlow itself. Misconfigured
deployments, third-party API key exposure, or self-hosted infra issues are out of scope.

## No SLA

This is an open-source project maintained on a best-effort basis. We make no
guarantees regarding response or patch timelines.
